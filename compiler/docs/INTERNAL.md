# Lava™ 编译器内部结构

本节详细阐述 Lava™ 编译器的核心架构、关键组件及其多线程编译流程，旨在提供对其内部运作机制的深入理解。

## 一、解析核心 (Parsing Core)

Lava™ 编译器的解析流程通过一系列专门的组件完成，实现从源代码到抽象语法树（AST）乃至最终字节码的转换。

### 1. 类语法解析器 (`LavaCompileUnit`)

`LavaCompileUnit` 继承自 `CompileUnit`，负责执行源代码的初步语法分析。
*   **结构识别：** 主要解析静态定义的结构，例如：类声明、字段定义、方法签名（名称、参数、返回值）、修饰符（如 `public`, `private`）、注解等。
*   **内容跳过：** 对于方法体内部的代码块，解析器通过括号匹配机制直接跳过，不做详细解析。
*   **任务封装：** 将诸如字段赋值语句、表达式等内部内容封装为“解析任务”（`ParseTask`），延迟其详细解析。

### 2. 类语义解析器 (`CompileUnit`)

在 `LavaCompileUnit` 完成基础语法解析后，`CompileUnit` 承担起语义层面的分析与协调工作。
*   **符号解析：** 解析导入表 (`ImportList`)，将短名称解析为全限定名。
*   **常量计算：** 计算字段的编译期常量值。
*   **方法处理：** 执行方法重载检测、继承关系分析等。
*   **装饰器处理：** 调用预处理阶段注解的处理函数。
*   **任务执行：** 运行此前由语法解析器封装的各类“解析任务”。
*   **代码生成：** 最终负责协调各阶段，启动代码生成过程。

### 3. 解析任务 (`ParseTask`)

`ParseTask` 代表一个待处理的代码片段，它包含了对词法分析器 (`Tokenizer`) 位置的引用以及解析结果的目标存储位置。
*   **任务种类：** 包括但不限于字段赋值语句、注解参数赋值语句等表达式解析任务，以及方法体和构造函数等方法解析任务。

### 4. 方法解析器 (`MethodParser`)

`MethodParser` 专注于方法的内部结构分析和部分优化。
*   **结构分析：** 分析控制流语句，如 `for`, `if`, `break`, `switch` 等，并适时调用表达式解析器 (`ExprParser`)。
*   **优化集成：** 在解析过程中集成部分代码优化，例如死代码消除和寄存器分配。
*   **即时代码生成：** `MethodParser` 采用即时（Just-In-Time）写入策略，不生成完整的方法层语法树节点。若需暂存方法层结构，则会将其写入一个新的 `MethodWriter`，并封装为代码块。

### 5. 表达式解析器 (`ExprParser`)

`ExprParser` 负责对表达式进行详尽分析，涵盖各种运算和逻辑结构。
*   **功能范围：** 包括加减乘除、函数调用、字段赋值、变量自增等。`if` 和 `switch` 语句在作为表达式使用时，会与方法解析器共享部分代码。
*   **解析阶段：** 表达式解析过程主要由以下三个阶段构成：
  1.  **预处理：** 检测表达式是否为变量赋值语句。
  2.  **算子序列处理：** 表达式由一系列连续的“算子（operator）”序列构成一个“子表达式”。每个算子可能根据类型向前或向后关联。
    *   **前缀算子：** 例如前缀自增 (`++i`)。
    *   **值生成算子：** 例如立即数、数组字面量等。
    *   **值转换算子：** 例如方法调用 (`obj.method()`)、字段访问 (`obj.field`)、圆括号分组 (`(expr)`)。
    *   **终结算子：** 例如后缀自增 (`i++`)、`instanceof` 运算符、三目运算符 (`condition ? true : false`)。
  3.  **连接符处理：** 包含所有二元运算符（如 `+`, `-`），用于连接下一个子表达式；还包括分号或逗号等，用于结束解析流程。
*   **优先级处理：** 解析结束后，`ExprParser` 会根据运算符优先级从高到低对算子列表进行整合，使用“二元运算”算子将左右操作数合并为单一的算子表示，构建表达式树。

## 二、编译器核心组件与API (`Package Structure`)

此部分详细介绍了 Lava™ 编译器内部的关键组件、API 接口及其组织方式。

### 1. 外部接口 (API)

Lava™ 编译器提供了一套可扩展的 API，以支持定制化开发。
*   **注解处理器：** 允许开发者编写自定义注解处理器，在编译期间对特定注解进行处理。
*   **自定义语法 API：** 提供了在全局编译上下文中修改运算符优先级表的能力。或者，可以通过 `StateMap` 机制，在遇到已注册的自定义 Token 时触发回调函数，将当前的 AST 和 `Tokenizer` 传入，并返回新的 AST，从而实现自定义语法扩展。这些修改将在创建新的表达式解析器时生效。
*   **`StreamChain` API：** 允许编译器在编译期对链式方法调用（例如 `a().doFoo().doBar().get()`）进行优化，根据所有函数的名称和参数生成优化的字节码，从而消除中间对象，提高运行时效率。

### 2. ASM 代码生成核心类

Lava™ 编译器使用 RojASM 库进行字节码生成，以下是支撑代码生成的几个核心抽象。
*   **方法生成器 (`MethodWriter`)：** `MethodParser` 在解析方法体时直接调用 `MethodWriter` 来生成字节码。它采用分段的“标签” (`Label`) 机制来处理指令偏移量。
*   **标签 (`Label`)：** 标签用于表示字节码中的一个特定代码位置，通过 `[段索引, 偏移量]` 的形式进行精确引用。这种机制确保了在中间代码生成过程中，即使代码块长度发生变化，标签的引用位置依然稳定。
*   **代码块 (`Segment`)：**
    *   **追加限制：** 代码块仅允许在末尾追加代码（特别是 `SolidSegment`），已写入的部分不可更改。
    *   **长度可变：** 其他类型的代码块只能改变总长度，且标签只能指向偏移量0。
    *   **引用稳定性：** 这种设计保证了标签所引用的代码位置在整个生成过程中是稳定的。

### 3. 抽象语法树 (AST - Abstract Syntax Tree) 与中间表示

AST 是编译器核心的数据结构，用于表示源代码的结构。
*   **解析任务 (`ParseTask`)：** 前文已述，作为尚未解析的语法片段表示。
*   **方法解析器 (`MethodParser`)：** 方法解析器采用即时（JIT）写入方式，**不存在**方法层结构对应的 AST 节点。若需对方法层结构进行暂存，`MethodParser` 会将其写入新的 `MethodWriter`，并将整个 `MethodWriter` 打包为代码块(`Segment`)。
*   **表达式解析器 (`ExprParser`)：** 用于解析和构建表达式的 AST 节点。
*   **表达式语法树节点 (`ExprNode`)：** 构成表达式 AST 的基本单元。
*   **变量初始化状态管理器 (`VisMap`)：** 全称 **V**ariable **I**nitialization **S**tate **Map**。
    *   **状态跟踪：** 负责保存和计算局部变量的赋值状态。
    *   **错误检测：** `ExprParser` 和 `MethodParser` 在变量赋值和读取时会调用 `VisMap` 的相应方法，从而检测出读取未赋值变量等错误。
*   **寄存器映射计算器 (`VarMapper`)：**
    *   **分配策略：** 目前基于变量的实际生命周期区间进行分配。
    *   **优化级别：** 编译至字节码时，不考虑 SSA (Static Single Assignment) 等激进优化，以避免可能阻碍 JVM 即时编译器 (JIT) 自身的优化能力。
    *   **与 Javac 对比：** Javac 采用形式生命周期（在整个作用域内有效），而 `VarMapper` 采用实际生命周期，能更有效地利用局部变量槽位。

### 4. 核心类型系统 (`Core Type System`)

在标准 RojASM 类型系统之外，Lava™ 编译器引入了额外的 `Wildcard` 实例类型，以支持更高级的类型推断和联合类型处理。

```java
public enum Wildcard {
    // ... 其他标准类型

    /**
     * **有界通配符 (Bounded Wildcard)**
     * 表示带有交集上界（例如，{@code ? extends A & B}）的通配符类型。
     * 可以向上转型为其任一上界类型，但向下转型为该类型要求源类型满足**所有**上界。
     * 代表多个上界通过 {@code &} 连接（交集类型）。
     * <p>
     * **类型转换规则：** 对向下转型检查所有上界；向上转型则只要源类型匹配任一上界即可成功。
     * </p>
     * **示例：** {@code ? extends Number & Serializable} – 接受同时是 {@code Number} 和 {@code Serializable} 子类型的实例。
     */
    BOUNDED_WILDCARD,

    /**
     * **捕获通配符 (Captured Wildcard)**
     * 代表在泛型操作过程中推断或“擦除”的类型。
     * 实际运行时类型通常为 {@code Object}，但编译时允许直接转换为推断出的类型。
     * 用于方法调用或 Lambda 捕获中的通配符。
     * <p>
     * **类型转换规则：** 允许“自由”地向下转型为捕获到的可视/推断类型（例如，从 {@code Object} 向 {@code String} 转型，如 {@code List<String>.get()}）。
     * </p>
     * **示例：** 泛型 {@code List<T>} 中 {@code list.get(0)} 的返回类型，编译时捕获为 {@code T} 但实际写入为 {@code Object}。
     */
    CAPTURED_WILDCARD,

    /**
     * **联合类型 (Union Type)**
     * 表示类型可以是其 alternatives 中的任何一种（异或“或”类型）。
     * 不能直接从该类型向下转型（因为没有单一源类型匹配所有情况），但任何 alternative 类型都可以向上转型为它。
     * <p>
     * **类型转换规则：** 如果源类型能够转型为**任一**上界，则向上转型成功；向下转型则需要检查所有可能性（通常在模糊时会报错）。
     * </p>
     * **示例：** {@code String | Integer} – 接受来自 {@code String} 或 {@code Integer} 的向上转型，但向下转型需要类型检查。
     */
    UNION_TYPE
}
```

### 5. 上下文管理 (`Context`)

Lava™ 编译器通过上下文对象管理编译过程中的各种状态和配置。
*   **全局编译上下文 (`LavaCompiler`)：**
    *   **负责创建：** 生成线程本地的编译上下文 (`CompileContext`)、词法分析器、方法分析器和表达式分析器。
    *   **全局资源：** 包含全局符号缓存、解析缓存和短名称索引。
*   **本地编译上下文 (`CompileContext`)：**
    *   **线程隔离：** 包含大部分不支持多线程和递归的对象，例如方法分析器，供 `CompileUnit` 调用。
    *   **阶段特定状态：** 包含 `CompileUnit` 中仅在特定编译阶段有效的状态，以节省 `CompileUnit` 的内存空间。
    *   **对象复用：** 包含对象缓存，以复用 `Set`、`List`、`Map` 等常见数据结构实例。
    *   **嵌套管理：** 提供 `next()` 和 `prev()` 方法，用于处理嵌套类、Lambda 表达式、宏生成器等的解析。
*   **核心符号缓存 (`SymbolCacheLibrary`)：** 负责解析并缓存 JVM 运行时标准库的符号表。
*   **用户符号缓存 (`XXXLibrary`)：** 由用户提供的库文件（例如 `JarLibrary`、`FolderLibrary` 或自定义实现的库）所生成的符号表缓存。

### 6. 类型解析与推断 (`Resolution`)

此模块处理复杂的类型解析、方法重载选择和代码生成逻辑。
*   **解析缓存 (`LinkedClass`)：**
    *   **按需生成：** 这个类和它引用的`ClassNode`按需生成，从而节省内存。
    *   **Lambda 支持：** 判断哪些方法可以用于 Lambda 表达式。
    *   **继承链：** 生成一个类的所有递归父类列表，便于类型转换判断。
    *   **多态处理：** 按名称生成方法和字段的组件列表 (`ComponentList`)，并处理重载和桥接方法等多态相关行为。
*   **组件列表 (`ComponentList`)：**
    *   **方法/字段集合：** 存储具有相同名称的方法或字段的集合。
    *   **重载选择：** 根据输入的参数类型判断最终应执行哪个重载方法，或在无匹配时报错。
    *   **高级特性支持：** 支持方法参数默认值和命名参数调用等高级功能。
    *   **访问器生成：** 对于 JVM 11 之前的二进制兼容性，可自动生成 `access` 访问方法。
*   **方法推断 (`Inferrer`)：**
    *   **适用等级计算：** 根据泛型信息、可变参数 (`varargs`) 和输入参数等，计算一个方法的“适用等级”。等级最高的方法将被 `ComponentList` 选中。
    *   **编译时重载：** 包含一部分代码用于处理编译时的重载决议。
*   **Lambda 生成器 (`LambdaGenerator`)：** 根据情况选择使用 `invokedynamic` 或匿名类两种方式生成 Lambda 表达式。
*   **非静态类上下文 (`NestContext`)：** 管理非静态内部类或 Lambda 表达式对外部类实例 (`this`) 和外部变量的引用。
*   **导入列表 (`ImportList`)：** 管理所有类型的导入，包括单类导入、静态成员导入、全量导入、模块导入（WIP）以及导入限制 (`package-restricted`)。
*   **类型转换 (`TypeCast`)：** 负责判断两个类型之间是否可以进行转换，并返回可行的最差转换类型，从宽化（Widening）到窄化（Narrowing），再到完全不可行。

## 三、CompileUnit 多线程编译流程

Lava™ 编译器采用多阶段、部分并行的编译策略，通过细粒度的同步机制确保编译的正确性和效率。

### 第一阶段 (Pass 1)：类级别语法分析

此阶段主要进行初步的词法和语法分析，识别类的基本结构。
*   **包/模块信息：** 解析 `package` 声明、`module-info.java` 和 `package-info.java`。
*   **导入声明：** 解析 `import`、`import static`、`import any` 和 `package-restricted` 语句。
*   **访问修饰符：** 识别类的访问修饰符。
*   **嵌套结构：** 解析内部类 (`inner class`) 和辅助类 (`helper class`)。
*   **成员声明：** 解析方法、构造函数、初始化块 (`{}`)、静态初始化块 (`static {}`) 和字段声明。
*   **泛型信息：** 解析泛型参数。
*   **初步表达式：** 解析注解参数值和字段立即赋值中的表达式，但不进行符号解析 (`resolve`)。

> 线程同步

### 第二阶段 2.1 (Pass 2.1)：语义分析 - 名称解析

此阶段开始进行符号解析，将短名称解析为全限定名。
*   **继承/实现：** 解析 `extends`、`implements` 和 `permits` 语句中的类型名称。
*   **成员类型：** 解析字段类型、方法参数类型。
*   **方法签名：** 解析方法的完整签名。
*   **注解解析：** 解析各处的注解。

> 线程同步

### 第二阶段 2.2 (Pass 2.2)：语义分析 - 引用与关系

此阶段进一步分析类之间的引用关系和结构约束。
*   **循环引用检测：** 检测类之间的循环引用。
*   **泛型异常处理：** 处理泛型相关的异常。
*   **密封类检查：** 执行 `sealed` 和 `permits` 关键字相关的检查。
*   **方法定义冲突：** 检查方法定义是否存在冲突（包括继承和接口实现）。
*   **异常类型检查：** 判断 `throws` 声明中的类型是否可抛出（需要判断 `instanceof Throwable`）。此步骤需放在本阶段，因为依赖于类型互引用的解析。
*   **默认成员生成：** 生成默认构造函数、枚举类的 `values()`、`valueOf()` 和 `$VALUES` 字段，以及 `record` 类的 `hashCode()` 等标准方法。

> 线程同步

### 第二阶段 2.3 (Pass 2.3)：语义分析 - 方法覆盖与实现

此阶段专注于方法覆盖、接口实现和桥接方法的生成。
*   **`@Override` 注解处理：** 处理 `Override` 注解。
*   **接口实现冲突：** 判断接口方法是否存在实现冲突，从而强制当前类实现。
*   **父类实现检测：** 检查父类是否已实现当前类接口的方法。
*   **抽象方法检测：** 检测是否存在未实现的抽象方法。
*   **访问权限降级：** 检查方法覆盖时，访问权限是否被意外降低。
*   **桥接方法生成：** 生成泛型返回值或泛型参数相关的桥接方法。

> 线程同步

### 第三阶段 3.1 (Pass 3.1)：静态字段解析 (无线程同步)

**注意：此阶段与 3.2 阶段并行执行，不进行线程同步。**
*   **DFS 递归解析：** 采用深度优先搜索 (DFS) 递归解析 `static final` 字段的初始化表达式。
*   **循环引用警告：** 在遇到 `static final` 字段初始化中的循环引用时发出警告。

### 第三阶段 3.2 (Pass 3.2)：注解处理器执行 (无线程同步)

**注意：此阶段与 3.1 阶段并行执行，不进行线程同步。**
*   **注解解析与处理：** 解析所有注解，并将其交给对应的注解处理器进行处理。

> 线程同步

### 第四阶段 (Pass 4)：语义分析 - 方法体解析

此阶段进入方法体内部的详细语义分析。
*   **静态初始化块合并：** 合并所有静态初始化块中的代码。
*   **构造器初始化合并：** 合并所有构造器初始化块中的代码，并将其插入所有构造函数的前部。
*   **匿名类处理：** 解析和处理匿名类的结构和语义。
*   **`final` 字段赋值检测：** 检查所有 `final` 字段是否最终被赋值。

> 线程同步

### 第五阶段 (Pass 5)：最终序列化准备

此阶段进行代码生成前的最后准备工作。
*   **属性/成员清理：** 删除无需序列化到字节码的属性和成员，优化最终输出。
