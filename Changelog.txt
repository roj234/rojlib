20220223
==========================================
变量分配器/变量分配器X
  2020年初,我在破解一个混淆的插件时,发现它重用了不少变量id
  javac并没有这个功能,考虑到我需要做一个java编译器
  我先做出了它,并计划在KScript中测试

  功能:
    变量分配器:
      按照变量作用域分配变量ID,
      并在超出作用域后回收该ID以便复用,
      零分配 (除了第一次使用,不再new任何对象)
        上面这点我本来是想节省点内存,进而节省时间
        不过我发现GC并没有想象中那么慢
        不过不需要new对象还是不错的
    变量分配器X extends 变量分配器:
      按照执行流程分析变量的读写,进而:
        拆分变量作用域
        删除多余变量和/或其作用域
      eg:
        int i = 0;
        for(; i<123; i++) {
           // do something
        }

        i = 0;
        for(; i<123; i++) {
          // do something
        }

        int j = 0;
        j = 1;
      变量分配器X会将上下两个i视为不同变量并分配(可能)不同的ID
      也会取消对j的第一次写入

      实际上它所遵循的规则只有两条:
        1. 变量赋值后,赋值语句下方[没有语句能返回上方](前向跳转), 则将其视为新变量
        2. 变量在两次赋值间没有读取和前向跳转, 则前一次赋值可以放弃 (当然, 要保留side-effect)
